Guidance for uploading data

This guidance explains how to prepare your Excel file and avoid common errors when uploading data into the system.

Before you upload

Before starting any upload, make sure your file:

  • Is saved as a .xlsx Excel file
  • Contains one worksheet only
  • Has column headers in the first row
  • Has no blank rows above the headers
  • Does not contain merged cells, formulas, or hidden rows

Download upload templates

To reduce errors, use the correct template for the type of data you are uploading. Each template contains the required columns in the correct order.

  • VTM and originator reference template
  • Marketing authorisation (MA) upload template
  • SPC and product-level upload template
  • Reference price and comparator upload template
Information:

Do not change column names, remove columns, or reorder columns in the templates. Additional columns will cause the upload to fail.

Important

Column names must match exactly, including punctuation, brackets and capital letters.

VTM and originator reference data

Some uploads rely on Virtual Therapeutic Moiety (VTM) and originator information. This data is used to group products correctly.

Tables may include:

  • VTM name
  • Originator licensee
  • Originator brand name

How to avoid errors

  • Enter the VTM name only (do not include strength or form)
  • Use consistent spelling for VTMs across all rows
  • Use recognised company names for originator licensees
  • Avoid adding notes or brackets in name fields

Marketing authorisation (MA) data

MA uploads are reviewed before confirmation. The review table shows how the system has interpreted your data.

You will typically see:

  • VTM
  • MA date
  • Authorisation number
  • Product name
  • Company name
  • dm+d matched name

How to avoid errors

  • Use DD/MM/YYYY for all dates
  • Enter the full MA number (for example: MA/1999/2211)
  • Ensure product names match dm+d naming conventions
  • Use the legal company name exactly as authorised

SPC and product-level data

SPC uploads include detailed product and regulatory information. Errors here often relate to formatting or missing identifiers.

Review tables may include:

  • SPC date
  • SPC number
  • Product description
  • Product search key
  • Applicant name
  • Matched company

How to avoid errors

  • Enter SPC numbers in the correct column only
  • Leave alternate SPC number blank if not applicable
  • Ensure applicant name matches the submitting company
  • Avoid adding pack size notes into search key fields

Reference price and comparator data

Reference pricing uploads may include comparator prices, adjustment flags, and explanatory notes.

How to avoid errors

  • Enter prices as numbers (with or without the Â£ symbol)
  • Use a consistent format for flags (Yes / No)
  • Ensure comparator years are numeric (for example: 2022)
  • Only provide reasons for exclusion where relevant

Review and confirm your upload

After uploading, review the table carefully before confirming.

  • Check dates, names and numbers appear correctly
  • Confirm company and dm+d matches look correct
  • Cancel the upload if anything looks wrong

Once confirmed, uploads cannot be edited. Corrections require a new upload.

Common causes of upload errors

  • Incorrect column headers
  • Wrong date formats
  • Extra worksheets in the file
  • Hidden rows or merged cells
  • Free-text notes added to structured fields
  • Inconsistent naming across rows
